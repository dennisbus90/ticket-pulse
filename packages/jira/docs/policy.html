Privacy Policy

Last updated: 9 May 2026

This privacy policy describes how Zigge ("the app", "we") handles data when you use Zigge for Jira, a Forge app distributed via the Atlassian Marketplace. Zigge is operated by the developer behind the Atlassian Marketplace listing and runs entirely inside Atlassian's Forge runtime.

1. Summary

  • Zigge does not store, log, or proxy your ticket content.
  • When you click Analyze, the values of the ticket fields you have configured for analysis are sent directly from Atlassian's runtime to the AI provider you have configured (OpenAI or Anthropic), using your own API key.
  • Without a configured API key, no data leaves your Atlassian environment.
  • The only data Zigge persists is your encrypted API key reference and your field-mapping configuration, both stored in Atlassian's Forge app storage.

2. What data is processed

Zigge processes the following data in order to provide its features:

Data Source Purpose
Issue field values you select for analysis Jira (current issue) Sent to AI provider for the quality / estimation request.
Status changelog of the current issue Jira (current issue) Rendered locally in the Timeline tab. Not sent to any third party.
Your AI provider API key You (entered in Settings) Used to authenticate the AI provider call.
Your field-mapping configuration You (Settings) Determines which fields are analyzed and which is used for estimation.

3. Where data is sent

Zigge makes outbound HTTPS requests to one of the following endpoints, depending on which API key you have configured:

  • api.openai.com — when an OpenAI API key is configured. See OpenAI's privacy policy and enterprise data handling.
  • api.anthropic.com — when an Anthropic API key is configured. See Anthropic's privacy policy and privacy center.

Requests are made directly from Atlassian's Forge runtime to the chosen provider. Zigge does not operate any backend service that intermediates these requests, and no copy of the request or response is stored by Zigge.

Your relationship with the AI provider is direct. Because requests go to your configured provider using your own key, the AI provider's privacy and data-retention terms govern that data once it arrives. Refer to their policies for retention periods, training-use opt-outs, and enterprise data-processing addenda.

4. What Zigge stores

Zigge stores the following inside Atlassian's Forge storage:

  • Your AI API key, written via Forge's encrypted storage API. The plaintext key is never written to logs or external services.
  • Your field-mapping configuration — the list of fields to analyze, the field used for estimation, and related preferences.

Zigge does not store any ticket content, any analysis results, any user identifiers, or any usage logs.

5. Permissions

Zigge requests the following Atlassian Forge scopes (declared in the app manifest and surfaced on the install consent screen):

  • read:jira-work, read:issue-details:jira, read:project:jira — to read the issue field values you choose to analyze.
  • write:jira-work — to apply "Use this" suggestions back to the issue when you click the button.
  • storage:app — to store your API key reference and field configuration in Forge storage.
  • External fetch to api.openai.com and api.anthropic.com, for the AI request itself.

6. Data retention & deletion

Zigge stores configuration only as long as the app is installed on your Jira site. When you uninstall the app, Atlassian removes the app's Forge storage automatically.

Data sent to the AI provider is governed by that provider's retention policy. To delete prompt history at the provider, use the controls in your OpenAI or Anthropic account.

7. Children's data

Zigge is a workplace productivity tool and is not directed at children under 16. We do not knowingly process data from children.

8. Changes to this policy

We may update this policy as Zigge evolves. Material changes will be reflected in the "Last updated" date at the top of this page and announced via the Atlassian Marketplace listing's release notes.
