Zigge — AI insights for Jira tickets

Zigge adds an AI-powered panel to every Jira issue. It reviews ticket quality against the fields you care about, suggests an estimation value, and shows a clean status timeline — all without leaving the issue view.

1. Install & first run

Zigge installs as a Forge app from the Atlassian Marketplace. Once installed, open any Jira issue — the Zigge panel appears in the issue's side column.

On first open you'll see an empty state prompting you to configure AI analysis. Click the gear icon (top-right of the panel) or Open settings to begin.

2. Configure an AI API key

Zigge uses your own AI provider key — you bring the key, you pay the provider directly, and your data is governed by their terms. Zigge supports two providers:

  • OpenAI — get a key at platform.openai.com/api-keys.
  • Anthropic — get a key at console.anthropic.com.

In Zigge's settings, click Add API key, choose the provider, paste the key, and pick the model you want to use. You can store multiple keys and switch between them.

Where keys are stored. Keys are saved in Atlassian's encrypted Forge storage scoped to your app install. They never leave your Atlassian site except when Zigge calls the provider's API on your behalf with that key.

3. Quality reviews

On the Quality tab, configure which Jira fields Zigge should analyze (Settings → Analysis fields). Pick any combination of summary, description, acceptance criteria, or other custom fields.

When you click Analyze ticket, Zigge:

  1. Reads the configured fields from the current issue.
  2. Sends them to your chosen AI provider with a structured prompt.
  3. Returns a per-field score and severity-tagged findings.

For each finding, Zigge often offers a "Use this" suggestion — an improved version of the field's content you can apply with one click. The improvement is written back directly to the Jira field.

4. Estimation hints

On the Estimation tab, point Zigge at one Jira field that represents your estimation (Settings → Estimation field). Common choices:

  • Story Points (numeric)
  • T-shirt size (single-select dropdown)
  • Custom estimation field

Zigge analyzes the ticket alongside similar past tickets in the same project and suggests a value, with a short explanation of the reasoning. The suggestion is informational — it is not auto-applied to the field.

5. Status timeline

The Timeline tab shows every status the ticket has moved through, with the duration spent in each status. Use it to spot tickets that are stuck in review, bouncing between statuses, or waiting on someone.

Timeline data is read from Jira's changelog — no AI call is made for this view.

6. Data & privacy

Zigge sends ticket field content to your configured AI provider when you click Analyze. Without an API key, no data leaves your Atlassian environment. Zigge itself does not store, log, or proxy ticket content.

Read the full privacy policy for what is sent, where, and how it is governed.

7. Troubleshooting

"Add at least one field to analyze"

Open Settings and add at least one field under Analysis fields. Until you do, the Quality tab can't run analysis.

"No estimation field configured"

Open Settings and pick a single field under Estimation field.

Analysis fails immediately

Most often this is an invalid or expired API key. Re-add the key in Settings. If that doesn't fix it, check the AI provider's status page — Zigge surfaces the provider's error message verbatim when the call fails.

"Re-analyze" is disabled on the Estimation tab

On the Estimation tab, the Re-analyze action is intentionally disabled when Quality analysis fields are configured — run Re-analyze from the Quality tab instead, which covers both.
